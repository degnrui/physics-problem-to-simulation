选择整体构图重心

两种都满足你的状态机。区别在于首页与结果页中，输入区、会话区、runtime 的视觉占比和气质。

A. Quiet Studio

A. Quiet Studio

首页更安静，中心输入框绝对主导。结果页里中间会话区更稳，右侧 runtime 是高质量工作台，但不过度压场。

B. Active Workshop

B. Active Workshop

首页更像创作台入口，结果页里右侧 runtime 更有存在感，中间会话更像驱动预览的控制中枢。

我推荐 A 的原因

  • 更贴近“教师输入需求 -> 系统生成 -> 稳定打磨”的克制感
  • 首页不会滑向“半个结果页”
  • 结果页中聊天和 artifact 更容易保持专业、安静、可讲授的阅读节奏

B 的代价

  • 更容易把首页做得像轻工作台
  • 结果页若控制不好，会让 runtime 压过会话编辑逻辑
  • 更接近通用 AI 创作工具，而不是教学 studio
